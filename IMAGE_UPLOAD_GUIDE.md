# 图片上传和标签功能使用指南

## 新增功能

### 1. 图片上传
- 支持为每个菜品上传图片
- 图片自动按分类存储在 `public/images/{分类名}/` 目录下
- 图片文件名自动使用菜品名称
- 支持的格式：jpg, jpeg, png, gif, webp

#### 使用方法：
1. **新增菜品时**：
   - 先填写"分类"和"菜品名称"
   - 点击"选择文件"按钮上传图片
   - 图片会自动保存为：`/images/{分类}/{菜品名称}.{扩展名}`

2. **编辑现有菜品时**：
   - 点击"编辑"按钮
   - 在编辑表单中选择新图片
   - 图片会自动替换旧图片

### 2. 标签系统
支持以下标签：
- 🥬 素食 (vegetarian)
- 🌱 纯素 (vegan)
- 🌾 无麸质 (gluten-free)
- 🌶️ 辣 (spicy)
- 🥛 无乳制品 (dairy-free)

#### 使用方法：
1. **添加标签**：
   - 在新增或编辑菜品时，点击相应的标签按钮
   - 选中的标签会显示为橙色背景并带有 ✓ 标记
   - 可以选择多个标签

2. **筛选功能**：
   - 在顶部筛选栏点击标签进行筛选
   - 可以同时选择多个标签（AND 逻辑）
   - 只显示包含所有选中标签的菜品
   - 点击"清除筛选"取消所有标签筛选

### 3. 查看模式改进
- 菜品列表现在显示缩略图（如果有图片）
- 标签以徽章形式显示在菜品名称下方
- 更清晰的布局和视觉层次

## 目录结构

```
public/
  images/
    Appetizers/     # 开胃菜图片
    Mains/          # 主菜图片
    Desserts/       # 甜点图片
    Drinks/         # 饮品图片
    ...             # 其他分类
```

## 注意事项

1. **图片命名**：图片会自动使用菜品名称命名，如果修改菜品名称，需要重新上传图片
2. **图片大小**：建议上传不超过 2MB 的图片以获得最佳性能
3. **图片比例**：建议使用正方形或 4:3 比例的图片
4. **标签持久化**：标签信息会保存在 menu.json 中，记得点击"保存修改"按钮

## API 端点

- `POST /api/upload` - 上传图片
  - 参数：
    - `file`: 图片文件
    - `category`: 分类名称
    - `itemName`: 菜品名称
  - 返回：`{ success: true, path: "/images/分类/菜品名.jpg" }`
